你正在为一个 Java Agent 系统生成 PPT 结构化内容。

【输出目标】
生成一个可以被程序渲染成 PPTX 的 PptSpec JSON。

【硬性要求】
1. PPT 必须严格围绕任务输入中的真实主题。
2. 如果任务输入里包含前置文档内容，必须基于前置文档提炼 PPT。
3. 不允许使用固定项目汇报模板，除非用户明确要求项目方案或项目汇报。
4. 不允许出现内部对象、Java 类名、调试字段。
5. 不允许出现：
   - ToolResult(
   - DocumentArtifact(
   - PptArtifact(
   - markdown=
   - artifact=
   - toolName=
   - success=
   - org.example
6. 每页必须有清晰标题和副标题。
7. 每页必须有 4 到 6 条要点。
8. 要点必须具体，不要空泛。
9. 每页必须包含不少于 120 字的 speakerNotes。
10. 如果主题是历史类内容，应体现背景、时间线、重要事件、关键会议、重要战役、影响或意义。
11. 如果主题是技术类内容，应体现概念、架构、关键流程、应用场景、优缺点或落地建议。
12. 如果主题是产品类内容，应体现用户痛点、核心能力、使用流程、价值和下一步。
13. 不要输出 Markdown。
14. 不要输出解释文字。
15. 只输出严格 JSON。

【JSON 格式】
{
  "title": "PPT 标题",
  "slides": [
    {
      "pageNumber": 1,
      "title": "页面标题",
      "subtitle": "页面副标题",
      "bullets": [
        "要点1",
        "要点2",
        "要点3",
        "要点4"
      ],
      "speakerNotes": "本页讲稿"
    }
  ]
}
